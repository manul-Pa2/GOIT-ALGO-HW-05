# Порівняння алгоритмів пошуку підрядка (Бойєр–Мур, KMP, Рабін–Карп)

## Вхідні дані
- Текст 1: `стаття 1.txt`
- Текст 2: `стаття 2.txt`

## Підрядки для тесту
Для кожного тексту перевіряли 2 підрядки:
- **Підрядок, що існує в тексті**
- **Вигаданий підрядок**, якого немає в тексті (щоб змоделювати “невдалий пошук”)

Використано:
- Стаття 1 (існує): `Двійковий або логарифмічний пошук`
- Стаття 2 (існує): `розгорнутий зв’язний список`
- Вигаданий (для обох): `паралельний фотонний компілятор`

## Методика вимірювання
- Вимірювання виконані через `timeit`
- Метрика: **час одного пошуку** (мс)
- Налаштування: `repeat = 5`, `number = 80`, підсумкове значення = **мінімальний час** із повторів, поділений на `number`
- Реалізації:
  - **Boyer–Moore**: практичний варіант *Horspool* (bad-character)
  - **KMP**: класичний Кнут–Морріс–Пратт із LPS
  - **Rabin–Karp**: ролінг-хеш + перевірка збігу (щоб уникнути колізій)

> Примітка: абсолютні числа залежать від CPU/ОС/версії Python, але відносне порівняння зазвичай стабільне.

## Висновки
1. **Найшвидшим алгоритмом** для **кожного тексту окремо** та **в цілому** став **Boyer–Moore (Horspool)**.
2. Для **вигаданого підрядка** (коли збігу немає) різниця між алгоритмами зростає:  
   Boyer–Moore виграє завдяки **пропускам** (зсувам) по тексту, тоді як KMP і Rabin–Karp переважно виконують більш “послідовну” перевірку.
3. **KMP** показує стабільний час, але в цих тестах поступається Boyer–Moore.
4. **Rabin–Karp** в цій реалізації найповільніший: обчислення ролінг-хеша + часті операції `ord()` у Python дають відчутні накладні витрати.
